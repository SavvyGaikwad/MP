Core Architecture

Four Layers of Controlled Intelligence

Every query passes through a purpose-built pipeline designed to guide cognition, not replace it.

01
Input Processing Layer

Intent Classification Engine

Every query is semantically tokenised, embedded, and classified into one of four academic domains. Domain-specific restriction strategies are activated automatically — the system knows exactly how much guidance is appropriate for maths vs essay writing vs conceptual learning vs code.

✍️ Essay Writing 🔢 Mathematics 💡 Conceptual 💻 Programming
02
Controlled Response Layer

Bloom's Taxonomy Depth Control

Responses are locked to Understand / Apply levels of Bloom's taxonomy. The depth controller and prompt engineering together prevent the model from reaching "Create" — ensuring you always do the synthesis yourself.

03
Constraint Engine

Response Filtering & RCE

A post-generation filter catches complete solutions, final answers, and compilable code even if the LLM tries to provide them. Token limiters, solution completeness detectors, and step-elimination modules run on every output.

04
Socratic Method

Question-Driven Scaffolding

Every response closes with a precision-crafted Socratic question. You receive key concepts, a thinking scaffold, and one question that forces you to reason one level deeper — not a passive reading experience.

05
LLM API Control

Deterministic Parameter Engine

Temperature locked at 0.3, Top-p at 0.8, with stop sequences that prevent full completion. Reasoning mode enabled for multi-step academic guidance that is consistent and non-hallucinatory.

06
Integrity Layer

Academic Integrity by Design

Not a bolt-on filter — integrity is baked into every layer of the architecture. GuidedMind cannot, by design, enable plagiarism, passive learning, or AI over-reliance.
